# Changelog

The AI games hardened this engine — each entry below is a defect a game surfaced, captured and fixed.

| Requirement | Title | Games | What broke |
| --- | --- | --- | --- |
| R12 | Win-by-paradox needs a procedure | [10](games/010/README.md) | Game 10 died on a self-contradiction between Rules 202 and 211 instead of pausing to resolve it. Built the paradox procedure: diagnose, safe-mode ballot, revert, stalemate backstop. |
| R40 | The judge rules on form, not legality | [16](games/016/README.md) | The GM rejected a perfectly legal motion as 'ruled out of order.' classify() was narrowed to judge well-formedness only; legality is a post-vote question (Suber Rule 104). |
| R108-numbering | Proposals number from 301, engine-owned | [16](games/016/README.md) | A rule was double-numbered (shown as both 214 and 305). Proposal numbering became engine-owned, counting from 301 whether or not a motion is adopted. |
| R52 | Cite or refuse: no conjured awards | [17](games/017/README.md) | The GM conjured a 120-point grant no rule authorized. GM score/threshold values must now cite an existing rule; uncited awards are stripped and the refusal is published. (The judiciary overturned the live grant 2-1.) |
| R61 | A refused award is public, not silent | [18](games/018/README.md) | A stripped score-delta vanished with no trace. It became a public event, so a reader can see what the engine refused and why. |
| R62 | An independent game auditor | [18](games/018/README.md), [19](games/019/README.md) | Reviewing a game by eye missed violations. Built audit_game.py to independently re-derive tallies and check every cited authority — it found a previously-undetected +75 violation in game 16 on its first run. |
| R53 | A bad verdict degrades, never crashes | [21](games/021/README.md) | Game 21 crashed at turn 29 when the GM's verdict tried to repeal a nonexistent rule. An invalid verdict now degrades to proposal-applied-no-awards instead of killing the run. |
| R33 | Seat outages are handled, not fatal | [22](games/022/README.md), [23](games/023/README.md), [24](games/024/README.md) | Cross-model games died on rate-limit (429) and connection errors. Honored Retry-After, distinguished 429 from 5xx, and made a seat that cannot act a defined, public, three-strike consequence. |
| R73 | The voting rule must cite its authority | [27](games/027/README.md) | At turn 63 the GM declared 'unanimous consent' citing no rule, and the engine ran the vote anyway. The engine now refuses an uncited voting rule up front, retry-then-halt. |
| R74 | A no-award verdict says so | [27](games/027/README.md), [28](games/028/README.md) | An adopted change with no score effect was indistinguishable from a silently-stripped one. A verdict with no awards now declares it, ending the ambiguous 'missing_award' audit class. |
| R75 | The GM checks who is eligible | [27](games/027/README.md) | Rule 310 barred negative-score players from voting, yet all three played on for ~85 turns while deeply negative — the GM never enforced it. The procedure prompt now asks, each turn, who is ineligible under the rules in force. |
| R76 | Skipped turns print | [26](games/026/README.md), [29](games/029/README.md) | A turn whose proposer went absent showed a blank heading on the console. It now prints the seat, the reason, and 'turn skipped,' matching the site. |
| R77 | Adopted text performs its own surgery | [28](games/028/README.md) | The table adopted 'Rule 209 is repealed and replaced' three times as bare additions — 209 was never actually repealed, so the statute held four contradictory caps. The GM now translates a rule's own textual surgery into real repeal/amend operations. |
| R78 | The procedure retry carries feedback | [29](games/029/README.md) | Game 29 died at turn 2: the engine re-asked a refused procedure with the identical prompt, and a temperature-0 GM returns the identical refused answer. The retry now feeds back the violation so the GM can correct it. |
| R79 | The auditor flags a drifting reading | [32](games/032/README.md) | The GM's voting-rule reading changed between votes with no rule change to explain it. The auditor now flags an unexplained procedure-consistency drift it previously couldn't see. |
| R80 | Null-tolerant GM parsing | [31](games/031/README.md) | The GM gave a perfect reading but wrote value:null where value was meaningless; the parser only defended against absent keys, crashed, and scored a well-formed answer as broken. Explicit null is now treated like an absent key. |
| R81 | The fallback declares, never fabricates | [31](games/031/README.md) | A parse failure silently returned a fabricated 'unanimous' procedure and the game ran a vote under a rule nobody wrote. A parse failure now declares itself unenforceable, routing into the retry-then-halt path. |
| R82 | Refuse before the vote, not after | [31](games/031/README.md) | A turn conducted a full vote under a procedure the engine should have already refused. The refusal gate now fires before any motion runs, not after the verdict. |
| R84 | The transmutation threshold stays in its lane | [32](games/032/README.md) | In a 46-rule statute the GM kept filling the ordinary voting rule with the transmutation threshold (Rule 109), wrongly defeating three majority-supported motions including the game's final vote. Field-discipline plus its own previous reading as continuity context. |
| R85-R87 | A player can claim a win the GM missed | [39](games/039/README.md) | Voltaire crossed the 120-point win threshold at turn 32, but no one — GM or player — noticed, and the game ran to the turn limit. Players can now declare a win (adjudicated + confirmed, never self-executing), and the auditor flags a reachable win left unclaimed. The new auditor immediately caught a second, previously-unknown case: game 33's Arendt met a 1-point threshold at turn 73. |
| R90 | The vote-weight ceiling now scales with the table | [41](games/041/README.md) | A healthy 5-player table adopted a rule giving one player vote weight 4, and the engine halted at turn 32 on a flat cap of 3 — an honest refusal, but the wrong number: the principled ceiling is one less than the seated player count, the largest weight that can never by itself force a majority. The bound is now table-relative and recomputed from the currently seated roster, so a seat change (an unseat, R59) can tighten it too. |
| R98 | Degenerate output can't crash the harness | [72](games/072/README.md), [73](games/073/README.md), [74](games/074/README.md) | Three games in a row died the same way, with no terminal event: a seat wrapped a rule number in an extra pair of brackets — targets [[301]] instead of [301] — and the proposal parser crashed the whole run trying to read the nested entry as a number. Malformed entries inside a well-formed list or table are now coerced when they honestly can be and dropped when they can't, so a garbled element degrades to absence — which the validators may then refuse in the open — instead of killing the game. |
