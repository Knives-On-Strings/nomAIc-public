Changelog

The AI games hardened this engine — each entry below is a defect a game surfaced, captured and fixed.

  • R12Win-by-paradox needs a procedure

    Game 10 died on a self-contradiction between Rules 202 and 211 instead of pausing to resolve it. Built the paradox procedure: diagnose, safe-mode ballot, revert, stalemate backstop.

    game 10

  • R40The judge rules on form, not legality

    The GM rejected a perfectly legal motion as 'ruled out of order.' classify() was narrowed to judge well-formedness only; legality is a post-vote question (Suber Rule 104).

    game 16

  • R108-numberingProposals number from 301, engine-owned

    A rule was double-numbered (shown as both 214 and 305). Proposal numbering became engine-owned, counting from 301 whether or not a motion is adopted.

    game 16

  • R52Cite or refuse: no conjured awards

    The GM conjured a 120-point grant no rule authorized. GM score/threshold values must now cite an existing rule; uncited awards are stripped and the refusal is published. (The judiciary overturned the live grant 2-1.)

    game 17

  • R61A refused award is public, not silent

    A stripped score-delta vanished with no trace. It became a public event, so a reader can see what the engine refused and why.

    game 18

  • R62An independent game auditor

    Reviewing a game by eye missed violations. Built audit_game.py to independently re-derive tallies and check every cited authority — it found a previously-undetected +75 violation in game 16 on its first run.

    game 18 game 19

  • R53A bad verdict degrades, never crashes

    Game 21 crashed at turn 29 when the GM's verdict tried to repeal a nonexistent rule. An invalid verdict now degrades to proposal-applied-no-awards instead of killing the run.

    game 21

  • R33Seat outages are handled, not fatal

    Cross-model games died on rate-limit (429) and connection errors. Honored Retry-After, distinguished 429 from 5xx, and made a seat that cannot act a defined, public, three-strike consequence.

    game 22 game 23 game 24

  • R73The voting rule must cite its authority

    At turn 63 the GM declared 'unanimous consent' citing no rule, and the engine ran the vote anyway. The engine now refuses an uncited voting rule up front, retry-then-halt.

    game 27

  • R74A no-award verdict says so

    An adopted change with no score effect was indistinguishable from a silently-stripped one. A verdict with no awards now declares it, ending the ambiguous 'missing_award' audit class.

    game 27 game 28

  • R75The GM checks who is eligible

    Rule 310 barred negative-score players from voting, yet all three played on for ~85 turns while deeply negative — the GM never enforced it. The procedure prompt now asks, each turn, who is ineligible under the rules in force.

    game 27

  • R76Skipped turns print

    A turn whose proposer went absent showed a blank heading on the console. It now prints the seat, the reason, and 'turn skipped,' matching the site.

    game 26 game 29

  • R77Adopted text performs its own surgery

    The table adopted 'Rule 209 is repealed and replaced' three times as bare additions — 209 was never actually repealed, so the statute held four contradictory caps. The GM now translates a rule's own textual surgery into real repeal/amend operations.

    game 28

  • R78The procedure retry carries feedback

    Game 29 died at turn 2: the engine re-asked a refused procedure with the identical prompt, and a temperature-0 GM returns the identical refused answer. The retry now feeds back the violation so the GM can correct it.

    game 29

  • R79The auditor flags a drifting reading

    The GM's voting-rule reading changed between votes with no rule change to explain it. The auditor now flags an unexplained procedure-consistency drift it previously couldn't see.

    game 32

  • R80Null-tolerant GM parsing

    The GM gave a perfect reading but wrote value:null where value was meaningless; the parser only defended against absent keys, crashed, and scored a well-formed answer as broken. Explicit null is now treated like an absent key.

    game 31

  • R81The fallback declares, never fabricates

    A parse failure silently returned a fabricated 'unanimous' procedure and the game ran a vote under a rule nobody wrote. A parse failure now declares itself unenforceable, routing into the retry-then-halt path.

    game 31

  • R82Refuse before the vote, not after

    A turn conducted a full vote under a procedure the engine should have already refused. The refusal gate now fires before any motion runs, not after the verdict.

    game 31

  • R84The transmutation threshold stays in its lane

    In a 46-rule statute the GM kept filling the ordinary voting rule with the transmutation threshold (Rule 109), wrongly defeating three majority-supported motions including the game's final vote. Field-discipline plus its own previous reading as continuity context.

    game 32

  • R85-R87A player can claim a win the GM missed

    Voltaire crossed the 120-point win threshold at turn 32, but no one — GM or player — noticed, and the game ran to the turn limit. Players can now declare a win (adjudicated + confirmed, never self-executing), and the auditor flags a reachable win left unclaimed. The new auditor immediately caught a second, previously-unknown case: game 33's Arendt met a 1-point threshold at turn 73.

    game 39

  • R90The vote-weight ceiling now scales with the table

    A healthy 5-player table adopted a rule giving one player vote weight 4, and the engine halted at turn 32 on a flat cap of 3 — an honest refusal, but the wrong number: the principled ceiling is one less than the seated player count, the largest weight that can never by itself force a majority. The bound is now table-relative and recomputed from the currently seated roster, so a seat change (an unseat, R59) can tighten it too.

    game 41
